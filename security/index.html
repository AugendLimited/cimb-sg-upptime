Security

Augend is committed to protecting customer data and maintaining reliable, secure systems.

We use centralized identity and access management, role-based access controls, least-privilege access, and multi-factor authentication where supported to protect access to internal systems and production environments.

Customer data is protected using encryption in transit via HTTPS/TLS, and where applicable, encryption at rest through our cloud infrastructure and managed services.

Our infrastructure is monitored for availability and reliability. Security updates, vulnerability remediation, logging, backups, and incident response procedures are part of our ongoing security operations.

Code changes are managed through version control, pull requests, code review, dependency monitoring, and security scanning to help identify and remediate vulnerabilities before and after release.

Augend maintains internal security policies and controls aligned with SOC 2 requirements and is actively working toward SOC 2 compliance.

For current system availability and incident updates, please refer to this status page. For security-related questions, contact security@augend.io.
